# Agent Dashboard v2 重构草案

## 目标
把首页从“分层导览页”改成“全局驾驶舱”。

核心原则：
- 总览页先给结论
- 详情页再给原因
- 运行页最后给证据

---

## 当前问题

### 1. 首页信息过散
用户打开后不知道先看哪里，重点不够集中。

### 2. 智能体状态不可扫读
需要读很多文字，不能一眼看出：谁在线、谁忙、谁卡住、谁空闲。

### 3. 总览页承担了过多“解释页面结构”的任务
这对真实使用帮助不大，反而稀释了关键状态信息。

### 4. 列表、详情、证据混在一起
导致页面复杂，但不高效。

---

## v2 新原则

### 总览页回答
现在发生了什么？

### 智能体页回答
这个智能体为什么是这个状态？

### 任务页回答
这个任务具体怎么推进？

### 运行页回答
底层运行证据是什么？

---

## 总览页 v2 信息结构

### A. 顶部：全局判断条
只保留 4 个最重要结论：
- 平台状态：正常 / 轻风险 / 异常
- 当前主责智能体
- 当前最关键任务
- 当前最高风险

要求：一眼看懂，不出现长段解释。

### B. 中部左侧：智能体状态总表
每个智能体一张短卡，只显示：
- 名称
- 状态：在线 / 忙 / 卡住 / 空闲
- 当前动作一句话
- 风险：无 / 高上下文 / 长时静默 / 启动中
- 一个“查看详情”按钮

目标：可扫读。

### C. 中部右侧：重点任务区
只显示 3 类任务：
- 当前最重要任务
- 当前最卡任务
- 最近推进任务

每条只给：
- 名称
- 状态
- 负责人
- 一句话原因
- 查看详情

### D. 底部：运行风险带
只保留高价值运行信号：
- 高上下文占用
- 启动中
- 长时静默
- 接口异常

这块不做大卡片，只做短条目。

---

## 智能体状态表达改法

当前：偏解释型

改成：偏结论型

示例：
- 龙龙｜忙
- 正在统筹当前任务
- 风险：无

- Builder｜卡住
- 最近没有形成有效推进
- 风险：长时静默

状态颜色建议：
- 忙：蓝 / 亮色
- 在线：青色
- 卡住：红 / 橙
- 空闲：灰蓝

---

## 页面跳转原则

### 总览页
统一使用“查看详情”

### 智能体页 / 任务页 / 运行页
作为二级深挖页存在，不再要求用户必须按固定顺序浏览。

---

## UI 方向

### 风格
继续保留现在的深色科技感，但减少块数、减少说明文字、提高状态标签权重。

### 页面布局
建议采用：
- 顶部全局判断条
- 两列主体
  - 左：智能体状态总表
  - 右：重点任务
- 底部横向风险带

这样首页更像“驾驶舱”，而不是“目录页”。

---

## 改造优先级

### P1
- 重做总览页信息架构
- 重做智能体短状态卡
- 把跳转统一改成“查看详情”

### P2
- 智能体页做解释层
- 任务页做推进层
- 运行页做证据层

### P3
- 再考虑动效、视觉微调、部署稳定性包装

---

## 成功标准
如果改完后，用户打开首页能在 3~5 秒内回答这四个问题，就算成功：
1. 现在整体正常吗？
2. 谁最重要？
3. 哪个任务最值得看？
4. 哪个地方最危险？
