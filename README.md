# Agent Dashboard

一个面向 OpenClaw 的多智能体监控台 MVP，用来实时查看智能体状态、会话、配置、缓存状态与基础使用情况。

## 当前版本定位

这是第一版可交付 MVP，目标不是把所有底层能力一次做满，而是先解决三个核心问题：

1. 页面能打开
2. 服务不假死
3. 用户能看懂当前是断连、刷新慢，还是正在显示缓存

---

## 功能说明

### 已完成

- 多智能体总览
- Agent 团队卡片视图（main / builder / planner / qa）
- 团队泳道视图（活跃中 / 近期活跃 / 空闲 / 异常）
- 会话列表
- Agent 详情面板
- 配置总览
- 基础使用情况统计
- 会话详情抽屉
- 同域 API 代理
- 加载态 / 刷新态 / 缓存态 / 错误态
- 空状态文案与骨架屏
- 后端缓存化状态采集（避免接口被慢命令阻塞）

### 当前已知限制

- 底层 `openclaw status` 在当前环境可能较慢，甚至超时
- 因此页面虽然稳定，但数据实时性仍受底层状态源影响
- 首次冷启动时，可能先看到降级数据或空数据，随后再逐步刷新

## 架构说明

### 前端
- Vite + React
- 页面端口：`4173`

### 后端
- Node.js 原生 HTTP 服务
- API 端口：`3456`

### 代理
- 前端通过 Vite proxy 将 `/api/*` 转发到 `3456`
- 浏览器侧统一请求同域 `/api/...`

### 状态采集策略
- 后端后台异步刷新状态
- 请求接口时优先返回缓存
- 若刷新失败，保留最后一次成功缓存
- 若首次无缓存，接口立即返回降级结果，不阻塞页面

## 本地运行

### 1. 安装依赖

```bash
cd /root/.openclaw/workspace/apps/agent-dashboard
npm install
```

### 2. 启动后端 API

```bash
npm run dev:api
```

默认监听：

- `http://localhost:3456`

### 3. 启动前端页面

```bash
npm run dev -- --host 0.0.0.0 --port 4173
```

默认访问：

- `http://localhost:4173`
- 局域网/服务器地址：`http://<your-host>:4173`

## API 说明

### `GET /api/health`
返回后端服务与缓存刷新状态。

示例字段：

```json
{
  "ok": true,
  "service": "agent-dashboard-api",
  "cacheUpdatedAt": "2026-03-19T03:00:00.000Z",
  "refreshing": false
}
```

### `GET /api/overview`
返回仪表盘总览数据。

包含：
- summary
- usage
- agents
- recentSessions
- status
- config
- cache 状态字段

### `GET /api/agents`
返回 Agent 列表。

### `GET /api/agent/:id`
返回单个 Agent 详情与最近会话。

### `GET /api/refresh`
触发一次后台刷新。

## 页面状态说明

页面右上角状态徽章会显示：

- **正在加载**：首次加载中
- **正在刷新缓存**：后台正在刷新状态
- **显示上次缓存**：刷新失败但仍有旧数据可显示
- **数据异常**：当前请求失败且没有缓存可用

### 如何判断问题类型

#### 真断连
常见表现：
- `/api/health` 无法访问
- 页面直接报错
- 代理返回 502 / 504

#### 刷新慢
常见表现：
- 服务在线
- 页面有缓存提示
- 数据不是最新，但不会空白假死

#### 底层状态源慢
常见表现：
- 页面能打开
- API 能快速返回
- 但数据更新频率低，或长期显示缓存

## 适合的使用场景
